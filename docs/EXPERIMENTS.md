## Выполнимо сейчас (софт + имеющиеся данные)

Приоритет по ценности×простоте:

1. **Сравнение с DIS-Flow и LK** (OpenCV `DISOpticalFlow`, `calcOpticalFlowPyrLK`).
   Прогнать на баге, сравнить траектории/resid. Позиционирует наш лёгкий блок-матчер
   против стандартов. Быстро, информативно. → **тастер сделан, см. results/bag_vs_optflow.png**

2. **RANSAC для global motion**: вместо медианы фитить глобальную трансляцию по блокам через
   RANSAC. → **ПРОВЕДЕНО, отклонён.** Шум −57% (0.56→0.24, находит плотный консенсус), НО
   яркость +38% (1.5→2.1): при дрейфе gain блоки разъезжаются, узкий tau цепляет неверный
   кластер. MAD даёт бóльшую часть выигрыша по шуму без поломки яркости. clean/outlier ≈.

3. **Пространственная карта надёжности (spatial reliability map)**: по всей
   последовательности бага усреднить quality/кривизну на каждой позиции блока → heat map.
   Гипотеза: края кадра хуже (виньетирование, дисторсия, out-of-field). Если да — взвешивать
   центр. Прямо на кэше, дёшево.

4. **Temporal filtering**: низкочастотный фильтр потока по времени (сгладить покадровый шум).
   Компромисс: убирает джиттер, но добавляет лаг. Тест на баге (resid + путь/эталон).

5. **Temporal drift analysis**: количественно — как накапливается ошибка позиции по времени
   (drift/сек, отдельно на зависании vs в движении). Частично сделано в §3 FINDINGS.

6. **Адаптивный размер блока (TILE_SIZE)**: крупнее блок = устойчивее матч, хуже
   разрешение/дороже. Требует параметризации хардкоженных 8 в SAD-функциях. Тест 8/12/16.

7. **Weighted SAD**: веса пикселей внутри блока (центр-взвешенный, или по градиенту).
   Отличается от отклонённого взвешивания агрегации — здесь вес ВНУТРИ SAD. Тест.

8. **Self-supervised калибровка FOV**: минимизировать resid к бортовой оценке по FOV.
   Ограничено: эталон сам OF-ведомый, не земля. Даст относительный FOV, не абсолютный.

## Нужны доп. данные / железо

- **Multi-camera fusion**: в баге есть `/cam_usb` + `/cam_usb0` (две камеры!) + `/odometry/image`.
  Можно пробовать фузию, но сложно и не факт что камеры смотрят вниз. Средняя выполнимость.
- **Self-supervised cam-IMU offset + temporal delay**: минимизация ошибки EKF2. Нужен
  настоящий независимый эталон (RTK-GPS) — в текущем баге его нет (OF инжектится как GPS).
- **Rolling shutter компенсация** (приоритет-3 плана): нужен реальный RS-сенсор + синхро-IMU.
