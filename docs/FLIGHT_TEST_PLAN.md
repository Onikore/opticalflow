# Полётная карта: какой баг записать для теста всей системы

Один выезд, три полёта (~35 мин суммарно). Закрывает все оставшиеся неизвестные:
одометрия на целевых высотах, map-matching против настоящего спутника, teach-repeat,
конверт манёвров, честность quality, финальная ошибка посадки.

## 0. Требования к записи (критично, без этого тест обесценивается)

1. **GNSS ВКЛЮЧЁН как независимая истина.** НЕ подменять его нашим/чужим OF (в прошлых
   багах gps_input был синтетикой от OF — сверяться было не с чем). RTK — идеально,
   обычный GPS — приемлемо. Наша система в этом полёте — пассивный пассажир.
2. **IMU на высокой частоте.** В прошлых багах `/mavros/imu/data` шёл ~4–11 Гц — этого
   мало для проверки синхронизации гиро. Включить raw-поток ≥100 Гц
   (`HIGHRES_IMU` / поднять `SR_*`/интервалы сообщений).
3. Камера: `/odometry/image/compressed` + `camera_info` на максимальной частоте
   (если может 30 fps — 30, не 8).
4. **Фиксированная экспозиция камеры**, если прошивка позволяет (наше правило №1).
   Если нет — хотя бы не менять освещение сцены резко.
5. Обязательные топики: image+camera_info, rangefinder, baro (static_pressure/rel_alt),
   attitude (quaternion), compass_hdg, gps raw+velocity, local_position, rc/out.
6. **Маркер на точке взлёта/посадки** (яркая плита/крест ~0.5 м) — для финальной
   сверки ошибки рулеткой и для precision-landing крошки.
7. Сохранить: фрагмент спутниковой подложки района (SAS.Planet/Google, макс. зум,
   с координатами углов), время/погоду/высоту солнца.

## Полёт 1 — «Калибровка и конверт» (~12 мин, поле с дорогой рядом)

Цель: шумовой пол, ошибка vs высота, потолки манёвров, знаки гиро.

1. **Земля, 30 с статики** (винты крутятся, дрон стоит) — биасы гиро/нули.
2. Взлёт, **зависание 60 с на h=3 м** над текстурой — шумовой пол в зоне VL53.
3. **Ступени высоты**: зависание по 30 с на **5 / 10 / 20 / 40 / 70 / 100 м**
   (вертикальный профиль: шум vs h, quality vs h, кросс-чек баро↔дальномер).
4. Спуститься на 50 м. **Квадрат 100×100 м** на 2–3 м/с, спокойно (базовая одометрия).
5. **Манёвры на 50 м** (конверт): разгон до 8–10 м/с и торможение ×2; резкие развороты
   yaw ~45–90°/с ×3; энергичные крены-наклоны (как «уклонение») ×3; змейка 20 с.
6. **Зависание 60 с на 50 м** (дрейф ховера на высоте).
7. Посадка **на маркер**.

## Полёт 2 — «Миссия 2 км» (главный, ~15 мин)

Цель: сквозной тест задачи — миссия в WGS84, map-matching, teach-repeat, ошибка посадки.

1. Маршрут **~2 км петлёй** через РАЗНЫЕ поверхности (важно!):
   участок вдоль **дороги** (якорь) → **поле** → **кромка леса** →
   **над лесом 300–500 м** (стресс без якорей) → выход на поле/дорогу → назад.
2. **h = 75 м**, крейсер **5–8 м/с**, автомиссия из QGC (сохранить .plan файл!).
3. В середине маршрута — **зависание 30 с** (кадры для ре-локализации).
4. Возврат: **половина по тому же коридору** (teach-repeat тест),
   **половина срезкой по прямой** через новый участок (тест «кратчайший путь домой»
   с пересечением исходящего коридора).
5. Финальный заход и посадка **на маркер**. Замерить фактическое смещение от маркера
   рулеткой (+ зафиксировать RTK, если есть).

## Полёт 3 — «Слепые зоны» (~5 мин, опционально но ценно)

Цель: честность quality на реальных вырожденных поверхностях.

Проходы на h=30–50 м на 3–5 м/с над тем, что доступно: однородное поле без структуры,
вода/пруд, свежий укос, гладкий асфальт-площадка, граница тени облака. По 20–30 с на
поверхность. Ожидание: quality падает честно, confident-garbage нет (или ловим где есть).

## Что мы получим из этого бага (маппинг на проверки)

| Запись | Закрывает |
|---|---|
| ступени высоты | ошибка vs h на 50–100 м (сейчас экстраполяция с 30) |
| квадрат + манёвры | конверт: потолок пирамиды/де-ротация на реальных 45–90°/с |
| ховеры 3/50 м | шумовой пол и дрейф зависания на целевой высоте |
| миссия 2 км + GNSS-истина | одометрия 1%? на длинном плече; **абсолютная** сверка впервые |
| спутниковый фрагмент + кадры | **кросс-сезонный map-matching** (последнее допущение) |
| коридор туда-обратно | teach-repeat на целевой высоте |
| срезка по прямой | «кратчайший путь домой» с ре-захватом коридора |
| посадка на маркер | финальная ошибка посадки — ответ на «≤10 м?» измерением |
| слепые зоны | quality honesty на реальных поверхностях |
| IMU ≥100 Гц | синхронизация гиро (temporal delay), знаки де-ротации |

## Мелочи, которые легко забыть

- Перед взлётом: фото линейки/маркера с известной дистанции (замер FOV камеры).
- Записать один баг НА ЗЕМЛЕ с чистым вращением дрона в руках вокруг каждой оси по
  10 с (знаковый тест гиро: чистое вращение ⇒ decoded v≈0).
- Ветер >8 м/с — перенести (ховер-тесты станут несравнимыми).
- Солнце: не снимать в узкий створ анти-солнечной точки (hotspot на траве).
