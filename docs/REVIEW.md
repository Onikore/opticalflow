# Критический ревью (Lead CV & Embedded) + проверка на реальном полёте

Ревью репозитория глазами CV/embedded-эксперта (PX4/DroneCAN/STM32), затем **каждый
проверяемый пункт прогнан на реальном баге** (`rosbag2_2026_06_09-01_30_17`, 103 м,
GPS-denied). Итог внизу: что подтвердилось, что баг опроверг.

## 1. Embedded & Memory Bottlenecks

| Локация | Проблема | Фикс для C/Cortex-M |
|---|---|---|
| `compute_flow_improved:278-281` — `np.pad` ×4 на кадр | 4 полные копии кадра (~40 КБ на 96×96 int32) per-frame | Статические буферы с бордюром / clamp-индексация без копий |
| `:261-262` — `astype(np.int32)` обоих кадров | 2 полные int32-копии (8× памяти vs uint8) | `__USADA8` принимает packed uint8; int32 только аккумулятору |
| `_coarse_vol_zsad` — float64 + per-window mean | float на грубой стадии, скрытый второй проход | Целочисленный ZSAD через SAT (integral image, int32) |
| `_highpass` — `scipy.uniform_filter` float32 | float-фильтр + 2 копии | Separable box через SAT, int32 |
| `np.median` ×5 на кадр | сортировки | insertion-sort ≤25 эл., медиана+MAD из одного массива |
| `sliding_window_view` (SAD-объёмы) | не переносится | вложенные циклы + `__USADA8`; cost-объём статический 324 Б |

## 2. CV Math & Logic Critique

| Компонент | Изъян | Фикс |
|---|---|---|
| SAD (fine) | не zero-mean: дрейф яркости до 1.5 px | фикс. экспозиция (SCCB) или ZSAD-fine через SAT |
| Субпиксель | 1D-сепарабельный tri-фит игнорирует кросс-члены; остаточный bias 0.088 px | 2D-квадрик по 3×3 (проверено ниже — НЕ дал выигрыша) |
| Пирамида | coarse-минимум не проверяется на уникальность | coarse peak-ratio (проверено ниже — вредит на нормали) |
| Rejection | uniqueness/λ_min OFF по умолчанию → confident-garbage на плитке | включить по умолчанию? (проверено ниже — НЕТ, только условно) |
| Де-ротация | гиро сэмплируется в момент кадра, не интегрируется по интервалу | `Σω·δt` между mid-exposure; калибровать delay τ |
| DroneCAN | байтовый лейаут DSDL не сверен; знак/оси гиро — конвенция симулятора | сверка с `dronecan/DSDL`; знаковый тест на железе |

## 3. Edge-Cases & Failure Modes (топ-5 для EKF2)

1. **Плитка/полосы** → q=255 + ошибка 4–6 px → EKF уводит позицию → uniqueness/λ_min-гейт **условно** (см. §5: по умолчанию нельзя).
2. **Низкий fps × манёвр** (7.8 fps + 43°/с = 10.6 px) → за диапазон → boundary_reject (есть) + fps ≥30 на железе.
3. **Дропаут дальномера** → `v=rate·h` масштабируется мусором → min-гейт h (есть в пайплайне; rate-limit по данным бага не нужен — спайков нет).
4. **Ступенька экспозиции** → SAD-минимум смещается при высоком q → фикс. экспозиция / high-pass.
5. **Ошибка знака/оси гиро в `rate_gyro_integral`** → EKF удваивает вращение (+1.6 м/с ложных на 90°/с) → знаковый тест на столе: чистое вращение ⇒ decoded v≈0.

## 4. Top-3 фиксы из ревью (до проверки)

1. Anti-garbage гейт по умолчанию (coarse peak-ratio + λ_min).
2. Гейт высоты + гиро-интеграция по интервалу в `flow_to_velocity`.
3. Целочисленный SAT-препроцесс (один integral image на кадр — ZSAD-fine + high-pass + суммы окон без float).

## 5. ПРОВЕРКА НА РЕАЛЬНОМ БАГЕ (что подтвердилось, что нет)

База = дефолт-пайплайн (pyramid + tri + MAD): resid **1.09 м**, valid 1647/1669.

| Пункт ревью | На баге | Вердикт |
|---|---|---|
| **A. ZSAD на fine-стадии** | resid 1.09→**1.05**, valid тот же | ✅ подтверждён (малый, но чистый плюс; в C — через SAT почти бесплатно) |
| **B. Гейты по умолчанию** (λ_min + coarse-ratio) | resid 1.09→**3.76**, valid 1647→1278 | ❌ **ОПРОВЕРГНУТ** — рушит нормальный полёт |
| — только λ_min>200 | 2.24, valid 1567 | ❌ вредит (режет слаботекстурные, но верные блоки) |
| — только coarse-ratio 1.25 | 3.88, valid 1352 | ❌ вредит сильнее (реальная текстура не имеет острых coarse-пиков) |
| — coarse-ratio 1.05 (мягкий) | 1.67 | ❌ всё ещё хуже базы |
| **C. 2D-квадрик субпиксель** | 1.11 vs 1.09 | ❌ не даёт выигрыша (кросс-члены не важны при глобальном движении), дороже |
| **D. Гейт высоты** | h<0.3 = 18% отсчётов (уже гейтится); \|dh/dt\| макс 2.1 м/с, спайков >3 м/с **нет** | ✅ min-гейт нужен (есть); rate-limit для этого дальномера избыточен |
| **E. Гиро-интеграция по интервалу** | непроверяемо: IMU в баге ~4 Гц (между кадрами ≤1 отсчёт) | ⚠️ теоретически верно, тестировать на железе (ICM-42688 = кГц) |

### Главный урок проверки

Ревью-рекомендация №1 («включить anti-garbage гейты по умолчанию») — **классическая
ошибка кабинетного ревью**: защита от редкого режима (повторяющаяся текстура), купленная
ценой постоянного налога на нормальный полёт (+150% resid). Реальные данные подтверждают
исходное проектное решение репозитория: uniqueness/λ_min — **условные флаги** (включать
над плиткой/полем), не дефолт. Пятый случай за проект, когда реальный баг корректирует
теоретически-красивую идею.

### Что реально принять из ревью

1. **ZSAD-fine через SAT** — единственный алгоритмический пункт, подтверждённый багом
   (1.09→1.05), и он же решает embedded-проблемы (без float/scipy/pad-копий). В C-порт.
2. **Embedded-фиксы §1** — все валидны (статические буферы, uint8+`__USADA8`, SAT,
   insertion-sort медиана) — это про порт, баг их не опровергает.
3. **Гиро-интеграция по интервалу + сверка DSDL + знаковый тест** — на этап железа.
4. Гейт высоты min (есть), rate-limit не нужен.

### Уточнённый Top-3 (после проверки)

| # | Фикс | Обоснование | Trade-off |
|---|---|---|---|
| 1 | SAT-препроцесс + ZSAD-fine | подтверждён (−4% resid) + убирает float/копии для C | +36 КБ RAM (SAT int32), +1 проход |
| 2 | Гиро-интеграция по интервалу + знаковый тест DroneCAN на столе | защита от failure mode №5 (удвоение вращения) | ~0 CPU, нужны timestamps |
| 3 | uniqueness/λ_min как **режимный** переключатель (по типу поверхности/quality-статистике), НЕ дефолт | плитка = confident-garbage, но дефолт-он рушит норму | логика режима + гистерезис |
