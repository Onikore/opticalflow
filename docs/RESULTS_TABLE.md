## 2. Темпоральные (в основном работа EKF)

| Техника | Что делает | Баг | Вердикт |

## 6. Валидация и анализ (не техники — исследования)

| Исследование | Ключевой результат |
|---|---|
| Реальное видео (FOV) | метрически состоятелен, FOV совпал на 2 клипах (расхождение метров = портретный FOV) |
| Снятое видео (надир) | 239/239 кадров, боковой дрейф 1.0% |
| Траектория из бага | форма до 1.1м на 109м, масштаб верный, **дрейф не копится** за 155с |
| Temporal drift | скорость роста ошибки ≈0; «путь 2×» = локальные петли, что возвращаются |
| Spatial reliability map | плохой только верх кадра (конструкция дрона), не виньетирование |
| Карта диапазона | search=4 → 124°/с, пирамида 295°/с (2.4×); объясняет «низко не кренится» |
| Fixed-point эмуляция | Q8-субпиксель + int SAD = float (Δ<0.0011px); int32 против переполнения |
| Rolling shutter | минорен для потока (shear сокращается; ускорение 0.5px) |
| Сравнение DIS/LK/Farneback/HS/фаза | наш 1.09м вровень с DIS(1.02)/LK(1.07); мы единств. int sparse под STM32 |

## Сводная статистика

- **Принято в дефолт:** 6 (median, parabolic, tri_subpix, pyramid, boundary, MAD) → clean −57%, шум −63%, ярк −80%, выбр −87% vs оригинал PX4Flow.
- **Опциональные флаги:** 6 (high-pass, census, uniqueness, peak_quality, fb, subpix_correction⚠️).
- **Отклонено с обоснованием:** 12 (RANSAC, curvature-weight, VMF, gradient-SAD, ZNCC, phase, weighted-SAD, gauss-subpix, spiral, EMA, spread-quality, gyro-centering).
- **Промежуточно/условно:** 6 (TILE_SIZE, 3D cost volume, temporal bilateral, structure tensor, multicam, multi-IMU, array).
- **Анализы/валидации:** 9.

**Главный сквозной вывод:** синтетика предлагает — реальный баг судит. 4 техники дали крупный
синтетический выигрыш, но не перенеслись (LUT +29%, VMF +6%, spread-quality, отчасти census).
Без проверки на реальном полёте половина «улучшений» была бы ложной.
